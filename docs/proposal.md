# 프로젝트 프로포절: 3D Music Explorer

## 3축 기반 시각적 음악 탐색 시스템

---

## 1. 문제 정의

### 1.1 현재 음악 추천 시스템의 한계

현재의 음악 추천 시스템(Spotify, Apple Music, YouTube Music 등)은 다음과 같은 근본적 한계를 갖고 있다:

**1) 블랙박스 추천 — 사용자가 통제할 수 없다**

기존 추천 시스템은 협업 필터링, 딥러닝 등 복잡한 알고리즘에 기반하여 "이 곡도 좋아하실 수 있어요"라고 제안한다. 사용자는 추천의 근거를 이해할 수 없으며, 원하는 방향으로 추천을 조절할 수단이 없다. "좀 더 밝지만 덜 강렬한 곡"을 원해도 시스템에게 이를 전달할 방법이 없다.

**2) 장르 기반 분류 — 음악의 실제 구조와 맞지 않는다**

음악은 전통적으로 장르(Rock, Jazz, Electronic 등)로 분류되어 왔으나, 장르는 음향적 유사성보다 문화적·역사적 맥락에 기반한 인간의 주관적 분류 체계이다. 같은 장르 안에서도 소리가 전혀 다른 곡이 존재하고, 다른 장르 간에도 음향적으로 유사한 곡이 많다.

**3) 필터 버블 — 새로운 음악 발견이 어렵다**

장르 기반 추천은 사용자를 같은 장르 안에 가두는 필터 버블을 형성한다. 사용자가 Rock을 많이 들으면 Rock만 추천되며, 음향적으로 유사하지만 다른 장르로 분류된 곡(Folk, Experimental 등)은 추천 대상에서 제외된다.

### 1.2 제안하는 해결책

**인간이 직관적으로 이해할 수 있는 3개의 축으로 음악을 시각화하고, 사용자가 3D 공간에서 직접 탐색하며 원하는 음악을 찾아가는 시스템.**

사용자는 다음과 같이 행동한다:
1. 3D 공간에 매핑된 음악들을 시각적으로 탐색한다
2. 3개의 슬라이더(밝기, 강도, 화성 복잡도)를 조절하여 원하는 방향으로 이동한다
3. 현재 위치 주변의 곡들을 미리 듣고, 마음에 드는 곡을 발견한다
4. 이 과정을 반복하며 자신만의 음악 취향 영역을 탐색한다

기존 "알고리즘이 추천해주는" 수동적 경험 → **"사용자가 직접 탐색하는" 능동적 경험**으로의 전환이다.

## 2. 연구 근거

### 2.1 연구 과정 및 데이터

본 프로젝트의 실현 가능성을 검증하기 위해 FMA(Free Music Archive) 데이터셋을 사용하여 6단계의 체계적 실험을 수행했다.

- **데이터셋**: FMA, 49,598곡, 518개 오디오 피처 (MFCC, spectral, chroma, tonnetz, ZCR, RMS)
- **분석 도구**: Python, scikit-learn, PCA, K-Means
- **검증 지표**: 분산 설명률, silhouette score, k-NN overlap/purity, ARI

### 2.2 핵심 발견

#### 발견 1: 장르 분류는 음향적 유사성과 맞지 않는다 (V1-V2)

518개 오디오 피처로 장르별 silhouette score를 측정한 결과 **-0.097 (음수)**. 이는 같은 장르의 곡들이 다른 장르의 곡들보다 음향적으로 더 가까이 있지 않다는 뜻이다. **장르는 소리가 아니라 문화에 기반한 분류**임이 정량적으로 확인되었다.

#### 발견 2: 음악의 자연 구조는 장르보다 단순하다 (V3)

비지도 클러스터링(K-Means)으로 발견한 2개의 자연 클러스터가 16개 장르보다 **231% 더 나은 분리도**를 보였다 (silhouette +0.247 vs -0.188). 음악은 에너지 수준에 따라 자연스럽게 두 그룹으로 나뉜다.

#### 발견 3: 클러스터 우위는 모든 세분화 수준에서 유지된다 (V4)

k=2-10 모든 클러스터 수에서 장르보다 우수한 성능을 보였다. k=10에서도 silhouette +0.135 (장르 -0.188 대비 여전히 압도적). 장르-클러스터 상관관계(ARI)는 항상 0.04-0.06으로 **장르와 음향 유사성은 근본적으로 다른 축**이다.

#### 발견 4: 3축 시스템은 탐색/발견에서 장르보다 4.7배 우수하다 (V5)

| 지표 | 장르 기반 | 3축 기반 |
|------|----------|---------|
| 장르 다양성 | 10.0% | **46.7%** |
| 카탈로그 커버리지 | 29.2% | **33.2%** |

장르 기반은 같은 장르 안에서만 추천하여 필터 버블을 형성하지만, 3축 기반은 장르 경계를 넘어 음향적으로 유사한 곡을 발견할 수 있다.

#### 발견 5: 3D 탐색형 공간은 현실적으로 구현 가능하다 (V6)

4가지 검증 테스트 결과:

| 테스트 | 결과 |
|--------|------|
| 축 해석 가능성 | **PASS** — 3축이 인간이 이해 가능한 음악적 특성에 대응 |
| 탐색 일관성 | **PASS** — 축을 따라 이동 시 장르가 점진적으로 변화 |
| 지역 일관성 | **PASS** — 근처의 곡이 다른 장르라도 음향적으로 유사 |
| 슬라이더 효과 | **PASS** — 각 축 조절 시 예측 가능한 변화 발생 |

### 2.3 3축의 음악적 의미

| 축 | 의미 | 낮은 값 | 높은 값 |
|----|------|---------|---------|
| **Axis 1** | Brightness / Energy | Classical, Folk, Old-Time | Electronic, Hip-Hop, Soul-RnB |
| **Axis 2** | Intensity / Loudness | Hip-Hop, Electronic | Old-Time, Folk, International |
| **Axis 3** | Harmonic Complexity | Rock, Spoken | Classical, Instrumental, Jazz |

이 3축은 65.4%의 분산을 설명하며, 탐색 목적으로는 충분한 해상도를 제공한다.

## 3. 시스템 설계

### 3.1 아키텍처 개요

```
┌─────────────────────────────────────────────────────────┐
│                    사용자 인터페이스                       │

│  │   (WebGL/Three.js)    │  │  - 3개 축 슬라이더        │  │
│  │   - 곡 점 렌더링       │  │  - 현재 위치 표시         │  │
│  │   - 마우스 회전/줌     │  │  - 주변 곡 목록           │  │
│  │   - 커서 마커          │  │  - 미리듣기 플레이어       │  │
│  └──────────────────────┘  └──────────────────────────┘  │
└─────────────────────────────┬───────────────────────────┘
                              │
┌─────────────────────────────┴───────────────────────────┐
│                    백엔드 서비스                          │
│  ┌──────────────┐  ┌──────────────┐  ┌───────────────┐  │
│  │ 오디오 분석    │  │  PCA 매핑     │  │  k-NN 검색     │  │
│  │ (librosa)     │  │  (3D 좌표)    │  │  (주변 곡 탐색) │  │
│  └──────────────┘  └──────────────┘  └───────────────┘  │
└─────────────────────────────┬───────────────────────────┘
                              │
┌─────────────────────────────┴───────────────────────────┐
│                    데이터 레이어                          │
│  ┌──────────────────────────────────────────────────┐    │
│  │  음악 데이터베이스 (메타데이터 + 오디오 피처 + 3D 좌표) │    │
│  └──────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────┘
```

### 3.2 핵심 기능

**1) 3D 음악 공간 시각화**
- 모든 곡이 3D 공간에 점으로 표시
- 장르별 색상 구분 (선택적 — 장르 없이도 사용 가능)
- 마우스/터치로 회전, 줌, 패닝
- 현재 위치를 나타내는 커서 마커

**2) 3축 슬라이더 조절**
- Brightness/Energy: 소리의 밝기와 에너지 수준
- Intensity/Loudness: 볼륨과 강렬함
- Harmonic Complexity: 화성 구조의 복잡도
- 실시간 업데이트: 슬라이더 이동 시 즉시 주변 곡 갱신

**3) 주변 곡 탐색**
- 현재 위치에서 가장 가까운 k곡 표시
- 곡 정보 (제목, 아티스트, 앨범)
- 30초 미리듣기 플레이어
- "이 곡에서 출발하기" — 클릭 시 해당 곡 위치로 이동

**4) 탐색 히스토리**
- 사용자의 탐색 경로 기록
- "이전 위치로 돌아가기"
- 자주 방문하는 영역 북마크

### 3.3 기술 스택

| 컴포넌트 | 기술 | 근거 |
|----------|------|------|
| 프론트엔드 3D 렌더링 | Three.js / Plotly.js | WebGL 기반, 50,000+ 포인트 렌더링 가능 |
| 프론트엔드 UI | React / Next.js | 반응형 UI, 실시간 슬라이더 업데이트 |
| 오디오 분석 | librosa (Python) | 업계 표준 오디오 피처 추출 라이브러리 |
| 차원 축소 | scikit-learn PCA | 검증된 선형 차원 축소, 해석 가능한 축 |
| 주변 곡 검색 | KD-Tree / Ball Tree | O(log n) 검색, 실시간 응답 가능 |
| 음악 스트리밍 | Spotify Web API / 자체 스트리밍 | 30초 미리듣기 제공 |
| 배포 | Vercel / AWS | 정적 사이트 + API 서버 |

### 3.4 데이터 파이프라인

```
새 곡 등록
  → librosa로 오디오 피처 추출 (518개)
  → 6개 피처 그룹으로 집약 (chroma, mfcc, rmse, spectral, tonnetz, zcr)
  → 사전 학습된 PCA 모델로 3D 좌표 계산
  → 데이터베이스에 저장
  → 실시간으로 3D 공간에 반영
```

## 5. 차별화 요소

### 5.1 기존 서비스와의 차별점

| | Spotify | YouTube Music | Pandora | **3D Music Explorer** |
|---|---|---|---|---|
| 추천 방식 | 협업 필터링 + DL | 협업 필터링 | Music Genome | **사용자 직접 탐색** |
| 사용자 통제 | 좋아요/싫어요 | 시청 기록 | 엄지 올림/내림 | **3축 슬라이더** |
| 시각화 | 없음 | 없음 | 없음 | **3D 공간 탐색** |
| 장르 경계 | 장르 기반 | 장르 기반 | 장르 기반 | **장르 무관** |
| 필터 버블 | 강함 | 강함 | 중간 | **없음** |
| 새로운 발견 | 약함 | 약함 | 중간 | **강함 (4.7배)** |

### 5.2 핵심 차별점

1. **인간 중심 설계**: 알고리즘이 아닌 사용자가 탐색의 주체
2. **직관적 인터페이스**: 3개 슬라이더로 누구나 이해 가능
3. **장르 무관**: 문화적 편견 없이 순수 음향 유사성 기반
4. **필터 버블 해소**: 장르 경계를 넘는 음악 발견 가능
5. **데이터 기반 검증**: 6단계 실험으로 실현 가능성 정량 검증

### 7.2 산업 관점
- **음악 추천 패러다임 전환**: 블랙박스 알고리즘 → 투명한 시각적 탐색
- **비주류 음악 노출 증가**: 카탈로그 커버리지 33.2% (장르 기반 29.2% 대비 향상)
- **사용자 체류 시간 증가**: 능동적 탐색은 수동적 소비보다 높은 참여도 유도
- **데이터 인사이트**: 사용자의 탐색 패턴에서 음악 취향의 새로운 차원 발견 가능

## 9. 결론

본 프로젝트는 **"음악 추천에서 인간의 통제권을 되찾는다"**는 비전을 가지고 있다.

6단계의 체계적 실험을 통해 다음을 정량적으로 검증했다:

1. **장르 분류는 음향적 유사성과 맞지 않는다** (silhouette -0.097)
2. **3개의 연속적 축이 장르보다 음악을 더 잘 분류한다** (silhouette +0.247 vs -0.188)
3. **3축 기반 추천은 장르보다 4.7배 더 다양한 음악 발견을 가능하게 한다**
4. **3D 탐색형 음악 공간은 기술적으로 실현 가능하다** (프로토타입 배포 완료)

기존 추천 시스템이 "당신이 뭘 좋아할지 우리가 알고 있다"라고 말한다면, 3D Music Explorer는 **"당신이 직접 찾아보세요 — 우리는 공간을 제공합니다"**라고 말한다.
